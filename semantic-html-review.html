Semantic HTML Review

Importance of Semantic HTML

Structural Hierarchy

Proper heading hierarchy (h1-h6) maintains content structure and improves accessibility:

Correct Usage:

<h1>Main Title</h1>
  <h2>Subsection</h2>
    <h3>Sub-subsection</h3>

Semantic vs Presentational

✅ Semantic Elements

  • <header>
  • <nav>
  • <figure>

❌ Presentational Elements (Deprecated)

  • <center>
  • <big>
  • <font>

Structural Elements

<header>

Defines the header section of a document or section

<header>
  <h1>Site Title</h1>
  <nav>
    <ul>
      <li><a href="#">Home</a></li>
    </ul>
  </nav>
</header>

<main>

Contains the main content of the document

<main>
  <article>
    <h2>Article Title</h2>
    <p>Main content here...</p>
  </article>
</main>

<nav>

Represents navigation links

<nav>
  <ul>
    <li><a href="#">Home</a></li>
    <li><a href="#">About</a></li>
  </ul>
</nav>

<figure>

Contains illustrations, diagrams, photos, or code listings

<figure>
  <img src="diagram.jpg" alt="System diagram">
  <figcaption>Figure 1: System Architecture</figcaption>
</figure>

Text-Level Semantic Elements

<em> - Emphasis

Marks text with stress emphasis

Never give up on <em>your</em> dreams.
Never give up on your dreams.

<i> - Idiomatic Text

Used for text in a different voice or mood

There is a certain <i lang="fr">je ne sais quoi</i> in the air.
There is a certain je ne sais quoi in the air.

<strong> - Strong Importance

Marks text with strong importance

<strong>Warning:</strong> This product may cause allergic reactions.
Warning: This product may cause allergic reactions.

<b> - Bring Attention To

Highlights text without conveying importance

The <b>SuperSound 3000</b> performed well in tests.
The SuperSound 3000 performed well in tests.

Description Lists

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>

Quotations and Citations

<blockquote>

Used for longer quotations

<blockquote cite="https://example.com">
  <p>The future depends on what you do today.</p>
  <footer>
    - <cite>Mahatma Gandhi</cite>
  </footer>
</blockquote>

<q> - Inline Quotation

Used for short inline quotes

As the author said, <q cite="https://example.com">momentum is everything</q>.

Technical and Special Elements

<code> and <pre>

Used for code snippets and preformatted text

<pre>
  <code>
    body {
      color: red;
    }
  </code>
</pre>

<time>

Represents dates and times

<time datetime="2025-05-17T20:00:00">May 17, 2025 at 8:00 PM</time>

<abbr>

Represents abbreviations

<abbr title="HyperText Markup Language">HTML</abbr>
HTML

Sub/Superscript

Used for mathematical notation and chemical formulas

H<sub>2</sub>O
x<sup>2</sup>
H2O x2

Best Practices

  • Use semantic elements over generic <div> elements when possible
  • Maintain proper heading hierarchy (h1-h6)
  • Use appropriate elements for their intended purpose
  • Include proper language attributes when needed
  • Provide context with title attributes where appropriate

Key Takeaways

  • Semantic HTML elements improve the structure, accessibility, and SEO of your web pages.
  • Each element has a specific purpose, so it's important to use them correctly to convey meaning and context.
  • Understanding these semantic elements helps create more meaningful and accessible web content.
