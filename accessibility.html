Web Accessibility

WCAG Guidelines (POUR)

  • Perceivable

    Information must be presentable to users in ways they can perceive.

    • Provide text alternatives for non-text content
    • Provide captions and alternatives for multimedia
    • Create content that can be presented in different ways
    • Make it easier for users to see and hear content
  • Operable

    User interface components must be operable.

    • Make all functionality available from keyboard
    • Give users enough time to read and use content
    • Do not use content that causes seizures
    • Help users navigate and find content
  • Understandable

    Information and operation must be understandable.

    • Make text content readable and understandable
    • Make content appear and operate in predictable ways
    • Help users avoid and correct mistakes
  • Robust

    Content must be robust enough to be interpreted by various user agents.

    • Maximize compatibility with current and future tools
    • Use proper HTML semantics

WAI-ARIA Roles and Attributes

ARIA Roles

<div role="alert">Important notification</div>
<div role="dialog" aria-labelledby="dialog-title">
  <h2 id="dialog-title">Dialog Title</h2>
</div>
<div role="tablist">
  <button role="tab" aria-selected="true">Tab 1</button>
</div>

ARIA States and Properties

<button aria-expanded="false">Show More</button>
<div aria-hidden="true">Hidden content</div>
<input aria-invalid="true" aria-describedby="error-msg">
<span id="error-msg">Please enter a valid email</span>

Live Regions

<div aria-live="polite">
  Content updated dynamically
</div>
<div aria-live="assertive">
  Important immediate updates
</div>

Assistive Technologies

  • Screen Readers

    Software that reads content aloud for visually impaired users

  • Screen Magnifiers

    Tools that enlarge screen content for users with low vision

  • Alternative Input Devices

    Including voice recognition and adaptive keyboards

Keyboard Accessibility

Focus Management

<!-- Use tabindex appropriately -->
<div tabindex="0">Focusable element</div>
<div tabindex="-1">Programmatically focusable</div>

<!-- Visible focus indicators -->
<style>
:focus {
  outline: 3px solid #4CAF50;
}
</style>

Skip Links

<a href="#main-content" class="skip-link">
  Skip to main content
</a>

<main id="main-content" tabindex="-1">
  Main content here
</main>

Best Practices

Semantic HTML

<!-- Use semantic elements -->
<nav>
  <ul>
    <li><a href="#">Navigation item</a></li>
  </ul>
</nav>

<!-- Instead of -->
<div class="nav">
  <div><a href="#">Navigation item</a></div>
</div>

Form Accessibility

<form>
  <label for="name">Name:</label>
  <input 
    id="name"
    type="text"
    aria-required="true"
    aria-invalid="false"
  >
  
  <fieldset>
    <legend>Preferences</legend>
    <input type="checkbox" id="email-optin">
    <label for="email-optin">
      Receive email updates
    </label>
  </fieldset>
</form>

Image Accessibility

<!-- Informative image -->
<img 
  src="chart.png" 
  alt="Sales chart showing 50% growth in Q2 2025"
>

<!-- Decorative image -->
<img src="decoration.png" alt="" role="presentation">

<!-- Complex image -->
<figure>
  <img src="diagram.png" alt="Network diagram">
  <figcaption>
    Detailed description of the network architecture
  </figcaption>
</figure>
